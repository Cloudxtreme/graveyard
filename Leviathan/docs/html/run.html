:: Running the Bot

      While double-clicking the Perl files will work to run the bot, it is not recommended practice. Run the batch file when you want to run the bot. The reason for this, is that if you run a Perl file and it crashes due to some kind of error, the DOS window will disappear immediately, not giving you any time to read it. The batch files will keep the window open.

Details on the Main Files:

Leviathan.pl (Leviathan.bat) - This is the DOS version of the main bot (the one that signs on your messenger bots). When running the bot for the first time, it is HIGHLY recommended that you use this one first, to see that the bots connect and work properly.

GUI.pl (GUI.bat, GUI Debug.bat) - After you know your bot runs with the DOS version, you may (optionally) move on to the Graphical User Interface. If you have trouble getting the GUI to work: don't worry about it, the DOS version works just fine!

Command-Line Arguments

The command-line arguments are as follows:

Leviathan.pl Arguments
/? - List the options. /m - Start the bot up in Maintenance Mode.
/s - Suppress Perl warnings.
/d - Enable Debug Mode.
/aim - Only run AIM bots.
/msn - Only run MSN bots.
/irc - Only run IRC bots.
/toc - Only run TOC bots.
/http - Only run HTTP bots.
/smtp - Only run SMTP bots.
/cyan - Only run CYAN bots.
/yahoo - Only run YAHOO bots.
/jabber - Only run Jabber bots.

GUI.pl Arguments
/? - List the options.
/s - Suppress Perl Warnings
/d - Enable Debug Mode
(the other commands can be accessed by choosing "Advanced Startup" from the Connection menu)
