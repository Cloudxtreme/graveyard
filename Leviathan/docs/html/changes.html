:: Change Log

Version 1.5.1 Patch

  • Bot Connection Mechanism patched: uses pure Perl instead of XML to define connections.
Version 1.5
  • Message Queues inverted. Code is executed before the time is spent. The time now is used as a "cool down" limit. In this manner, AIM bots will respond more quickly depending on how busy they are.
  • GUI has been reprogrammed.
Version 1.4
  • GUI Temporarily Suspended: Due to drastic changes in Leviathan.pl, the GUI hasn't been updated and is therefore suspended (however, the source "GUI.txt" is included if you want to edit it yourself).
  • Bot Settings Change: Instead of files in the "bots" folder, you now define all your bots in "config/connections.xml" - the downside is that you have to use XML instead of plain text. However, now it's easier to add mirrors. If you have 5 AIM bots that use the same reply data, and then decide to change their brain, you just have to make one change. Also, you can turn individual mirrors on or off (or turn the entire bot on or off).
  • Timestamps Subroutine Update: Time::Format is now being used, so strings such as "Weekday, Month d yyyy @ H:mm:ss A.M." will work. However, for backwards compatibility the old tags will still work as well (however, <day_year> has been removed).
  • Games Broadcast Handler Update: Now supports Yahoo, IRC, and Jabber users to participate in multiplayer games.
  • Yahoo Messenger listener added (still in beta; may not sign on if many other bots are also connected).
  • Games::Manager renamed to Games::Multiplayer::Manager by request of CPAN admins (also: CPAN Documentation on Games::Multiplayer::Manager)
Version 1.3
  • Graphical User Interface added.
  • Eliza Brain Update: New Eliza objects are created for each user of an Eliza bot, so when Eliza tries to bring up something from past conversation, it won't mix up users (even though that can be quite humorous).
  • User Profile Update: User profiles now only contain the required data initially (stars, points, temporary blocks, etc.) However, &profile_send will add new variables to a profile as it needs them, giving you unlimited range of variables. (Run "util/Levi1.2 Profile Updater.pl" to update user profiles from older versions of Leviathan).
  • Alpha Brain Updates: The Alpha brain (Chatbot::Alpha 1.32) now has support for topics, as well as many bug fixes (i.e. ways to break out of infinite loops or recursion errors).
  • Command Update: Commands marked as "Hidden" will not only be excluded from the menu, but the "usage" command will pretend as if the command doesn't exist.
  • Documentation: Links for registering new usernames for the messengers have been added to the "Setup Bots" page in the documentation.
Version 1.2
  • Added Listeners: TOC and SMTP. TOC is an AIM listener that uses Net::AIM (some people had problems with Net::OSCAR). The SMTP listener will enable you to set up e-mail auto-responders.
  • Reply Sentence-Splitting: The bots will now split up your sentences and give you a paragraph of replies in return (i.e. "Hello. What's up?" --> "Hey! Not much, you?").
  • HTTP "Files": A few special "files" on the HTTP listener have been added:
    • Accessing /blocked.cgi?user=SENDER-username will return whether or not they are blocked (in <blocked>:<why> format).
    • Accessing /warner.cgi?user=SENDER-username will return whether or not they're on the Warners List.
  • HTTP Dynamic Pages don't write temporary files anymore; the updated page codes are sent without actually sending files.
  • Commands Added:
    !warnlist - Administrator Only - Modify the warners list
    !warn - Botmaster Only - Issue an AIM warning
Version 1.1
  • Messenger capability is now less strict. The bot won't require messenger modules that aren't going to be needed.
  • MSN ClientCaps are now fixed.
  • Command indexing now prints a notification for each command indexed (much like it does for including subroutines and handlers).
  • Menu command update: The menu sorts categories more efficiently, so if you have 10 or more categories they will remain in numerical, not ASCII, order.
  • Reload command update: The !reload command now reloads startup configuration and substitution data.
  • Bot fonts will now revert to defaults if the font could not be found (don't need to worry about itty bitty fonts on AIM anymore!)
  • Timestamp bugs fixed (a request for <month_num> will return from 1 to 12, not 0 to 11).
  • Including module Games::Manager for multiplayer games ($chaos->{games} holds the object).
  • Bug fix in AIM Chatrooms' timestamps (<mins> now filters in correctly).
  • Minor bug fixes in the handler files.
  • Documentation - "low-fi.html" is now even "lower-fi" - consists only of navigation and content frames.

Version 1.0 -- Initial Release

  • AiChaos Leviathan was created as an improved replacement to CKS Juggernaut. These changes are mentioning the major differences between Juggernaut and Leviathan.
  • Bot data update: using "Client:" to identify the messenger no longer works, using "Listener:" is now used to define the messenger.
  • Client format update: The $listener has been dropped, and $client now is formatted as "SENDER-username" (i.e. MSN-name@domain.com, or AIM-screenname).
  • User data update: A few new keys have been introduced, including those used for temporary block systems. Also, there are 20 Custom Keys that you can use in commands, etc. for keeping your own types of data.
  • The Jabber listener has been introduced.
  • Message queueing now is per username, instead of having one global queue.
  • Alpha brain added.
  • Brains update: instead of having "<brain>_get" and "<brain>_load" files and subs, they are included in a single file. The returning hashref defines what the names of the subs are, so the bot can load them.
  • Every included file returns information about itself in a hashref, not just commands.
  • Commands - The !menu command is more dynamic; adding a category is as simple as creating a command that ties to that category. The !menu command generates the category list dynamically.
  • Main configuration file is now in Perl form, allowing you to set any variable and not being limited to the ones Juggernaut gives you.
  • New Blocking Method: Anytime the bot has to block somebody by itself, it uses temporary blocks. The block equation is (number of times blocked)2 * 15 minutes.
  • BlackList Update: The !blacklist command now takes a reason the user was banned; this reason is saved in the blacklist and returned to that user when they start a conversation (MSN).
  • Conversation Logs Update: Collective conversation logs are now kept in "logs/chat", and the top-level "logs" folder is used for smaller logs (i.e. alerts, warnings, etc.)
  • The $chaos hashref has been reorganized. Details:

    Added Top-Level Keys:
    $chaos->{version} - Current program version number.
    $chaos->{author} - Program's author.
    $chaos->{debug} - Debug Mode On/Off.

    Renamed/Moved Keys:
    $chaos->{_system} has been renamed to $chaos->{system}
    $chaos->{_users} has been renamed to $chaos->{clients}
    $chaos->{_data} has been renamed to $chaos->{data}
    Configuration data has been moved from $chaos->{_system}->{config} to $chaos->{config}
    Bot data has been moved from top-level keys into the $chaos->{bots} key.
