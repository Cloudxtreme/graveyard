:: Brains

      This page will explain the basics on the brains: how to include them in your bot, how a brain file is formatted, and how to write your own brains.

:: Using the Brains

Here are some details on the default brains. Each brain listed below will give a short description, what to set the "Brain:" value to (in your bot files), and what to set the "Replies:" value to.

Alpha Brain
Description: Latest creation of AiChaos, Inc. - a simple yet powerful reply language.
Author: AiChaos, Inc. - Copyright 2004
Using the Brain:
Brain: Alpha
Replies: point to a folder containing TXT or CBA text documents; each document is a reply set

Chaos Markup Language
Description: An enhanced CML parser for CML 2.0 (similar to AIML coding).
Author: AiChaos, Inc. - Copyright 2004
Using the Brain:
Brain: ChaosML
Replies: point to a folder containing CML files; each document is a reply set

Eliza Brain
Description: Perl implementation to the Eliza therapist bot brain.
Author: Michal Wallace
Using the Brain:
Brain: Eliza
Replies: you can leave this blank; Eliza doesn't use replies

Juggernaut Brain
Description: A basic parser for CML 1.0 - originally created for CKS Juggernaut bots.
Author: AiChaos, Inc. - Copyright 2004
Using the Brain:
Brain: Juggernaut
Replies: point to a folder containing CML files; each document is a reply set

Nexus Brain
Description: AiChaos' Nexus brain--a bot that learns how to chat by chatting!
Author: AiChaos, Inc. - Copyright 2004
Using the Brain:
Brain: Nexus
Replies: point to a TEXT DOCUMENT. The document does not need to exist initially.

:: Format of a Brain
Brain Folder
All brains should go into the "brains" folder.

Brain File Name
The file name should be the name of the brain, lowercase and without spaces, followed by the .PL extension.

Contents
The brain should contain at least one subroutine, for getting a reply. The "GET" subroutine receives a set of variables, see below. Also, if the brain requires initialization, you will need a subroutine for loading the brain.

Returned Hashref
Brains should return a hashref containing the following keys:
Type => 'brain',
Name => 'the brain\'s name',
Description => 'short description on the brain',
RequireLoading => 1 for yes, 0 for no,
LoadingSub => 'if RequireLoading = 1, define the loading sub\'s name here',
ReplySub => 'sub name for getting a reply',
Author => 'your name',
Created => 'creation date',
Updated => 'updated date',
Version => 'version number',

:: Reply Sub Variables
Received Variables

$brain - the name of the brain.
$self - messenger object of the current user.
$client - user's client ID.
$msg - user's FORMATTED message (uppercase, no puncuation, etc.)
$omsg - user's ORIGINAL message.
$sn - the bot's screenname.

Returned Variables

$reply - how the bot replied to the message.

:: Loading Sub Variables
Received Variables

$bot - the bot's screenname
$brain - the brain name
$dir - the reply file/folder that the brain uses (defined with the "Replies:" in a bot file)

Returned Variables

1;
Confirmation that the brain was loaded correctly.
